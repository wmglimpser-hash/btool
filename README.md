# 古籍活化工坊

> 让古籍内容更容易被准确传播，而不是被随意曲解

一个面向文博、教育、科普和短视频创作者的古籍阅读与内容转化工具。我们坚持：**传播不能脱离原文，内容必须标注来源**。

## 项目简介

选择一页古籍，系统基于原文结构生成可传播内容（内置书目为预设模板，手动输入为通用演示）：

- **来源可信链路**：每部内置古籍标注原文来源、版本、参考注本与可信状态
- **白话解读**：将古籍原文转化为通俗易懂的现代白话文
- **传播要点**：提炼适合社交媒体传播的核心亮点
- **小红书文案**：生成带话题标签的小红书风格读书笔记
- **公众号长文**：生成带标题、导语、原文引用的公众号文章框架
- **短视频分镜**：生成 60 秒短视频的横向时间轴分镜脚本，每镜标注原文依据
- **文创分享卡**：生成带出处水印的竖排社交媒体分享卡片，支持下载 PNG

## 反伪科普设计

本项目内置四层约束机制，防止古籍内容传播沦为伪科普：

1. **原文锚定**：白话解读与分镜脚本均标注对应原文
2. **出处溯源**：每部内置古籍明确版本、参考注本与时代背景
3. **可信度分层**：「考据」与「演绎」定性标签，不使用硬编码百分比
4. **来源可信链路**：区分「原文来源」「项目整理」「生成内容」，并提供可点击的公开古籍数字文本链接

## 古籍来源可信链路

本项目将每条内容区分为三个层次：

- **原文来源**：内置样例采用公开古籍数字文本（如中国哲学书电子化计划 ctext.org），并提供可点击链接供用户核验
- **项目整理**：公开古籍文本经项目方整理标点与分段，便于演示
- **生成内容**：白话解读、短视频分镜、传播要点、分享卡文案等均为系统基于原文结构生成的演示模板，不代表学术结论

手动输入模式会明确标注「用户输入 · 版本待确认」，要求用户补充来源名称、来源链接、版本说明与参考注本，并降低置信度至「待核验」。

## 使用方式

1. 在左侧书架选择一本典籍，或点击「输入古籍文本」粘贴自己的古籍内容
2. 在中间阅读器中选中一段原文
3. 点击工具条中的产物按钮（白话解读、传播要点、分镜脚本等）
4. 右侧面板自动展示生成内容，包含来源与可信度标注
5. 分享卡片支持「下载图片」和「复制文案」

## 技术栈

- 纯前端单页应用（HTML + CSS + JavaScript）
- 无需后端，可直接在浏览器中运行
- 使用本地 html2canvas 实现分享卡片导出（三重加载保障）

## 设计风格

新中式数字古籍风格 —— 融合传统水墨意境与极简现代布局，以"古籍今用，以今释古"为核心理念。
