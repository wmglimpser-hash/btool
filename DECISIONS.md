# 古籍活化工坊 · 设计决策

> 记录产品演进中的关键取舍，便于回溯"为什么这样设计"

---

## D1: 首页形态 — 工作台而非案例墙

- **决策**：首页直接呈现三栏工作台（书架+阅读器+产物面板），案例内嵌为右侧面板默认示例
- **原因**：
  - 案例展示页让用户需要滚动很久才能进入实际工具，转化弱
  - 用户一进来就能开始选文生成，符合"资源优先，主题辅助"
  - B方案（工作台风）优于A（展示）和C（案例墙）
- **日期**：1.3.6
- **状态**：已落地

---

## D2: 交互模式 — 逐个按需生成而非全量吐出

- **决策**：从「选书→全量活化」改为「阅读→选中片段→逐个点选产物」
- **原因**：
  - 全量生成内容过长，用户注意力分散
  - 逐个生成更符合短视频/社交媒体的创作习惯
  - 降低单次加载的信息压力
- **日期**：1.3.0
- **状态**：已落地

---

## D3: 可信度表达 — 定性标签而非百分比

- **决策**：使用「考据」/「演绎」定性标签，不再使用硬编码置信度百分比条
- **原因**：
  - 百分比暗示精确量化，但实际为预设模板无法真实计算
  - 硬编码数字会损害用户信任
  - 定性标签更诚实、更专业
- **日期**：1.3.2
- **状态**：已落地

---

## D4: 文案风格 — 去工程化、去 AI 化表述

- **决策**：移除所有「活化」字样，改为具体功能名；弱化"AI 生成"表述
- **原因**：
  - 「活化」被用户反馈为抽象、低质量感
  - 过度强调 AI 会降低内容可信度
  - 前端只展示"选中了什么→能得到什么"
- **日期**：1.3.0 ~ 1.3.2
- **状态**：已落地

---

## D5: 来源链路 — 首屏可见且可追溯

- **决策**：来源卡固定在阅读器顶部，包含原文来源链接、版本、参考注本、可信状态
- **原因**：
  - 反伪科普是产品核心价值之一
  - 可信信息必须在第一屏可见，不能折叠到深层
  - 内置古籍用 ctext.org 链接，手动输入标注"待核验"
- **日期**：1.3.6
- **状态**：已落地

---

## D6: 移动端策略 — 入口优先于内容完整

- **决策**：移动端首屏优先给入口（书目tabs + 阅读器），产物面板用底部抽屉而非页面内嵌
- **原因**：
  - 手机屏幕有限，产物面板在末尾会导致用户滚动极长距离
  - 底部抽屉模式符合移动端常见交互范式
  - 选中后自动滑出，减少操作路径
- **日期**：1.3.7
- **状态**：已落地

---

## D7: 技术栈 — 纯前端无框架

- **决策**：使用纯 HTML + CSS + JavaScript 单文件，不引入 React/Vue 等框架
- **原因**：
  - GitHub Pages 直接部署，无需构建步骤
  - 单文件便于版本管理和快速迭代
  - 无外部框架依赖，加载速度快
  - 参赛作品需保证可独立运行
- **日期**：1.0.0
- **状态**：维持中

---

## D8: 首页从"工作台优先"改为"案例驱动" — 取代 D1

- **决策**：首页首屏改为完整案例展示，不再以阅读器为第一视觉中心；阅读器保留为"进入创作"区域
- **取代**：D1（工作台优先）
- **原因**：
  - 参赛评审 5 秒内需看懂产品全貌，工作台需要操作才能看到产物
  - 案例驱动让评审和用户首屏即见"文字+图片+视频"三类产物全貌
  - 产物面板预载《关雎》完整案例，主题"《关雎》如何表达理想关系？"
  - 主题输入可刷新案例，绑定古籍资源，禁止无源生成
- **实现方式**：
  - 首屏=案例展示（主题横幅+来源卡+三类产物摘要+主题输入+展开详情）
  - 下方=阅读器工作台（"进入创作"入口）
  - API 预留四端点 + 四态 + demo 兜底（前端壳已就绪，真实生成未接入）
  - 文字/图片/视频三类产物当前为预设模板/CSS占位，标注"演示数据"
- **日期**：1.4.0（2026-07-04），1.4.1 安全加固与首屏压缩
- **状态**：已落地（前端壳+API预留，真实生成待接后端）
- **取代**：被 D9 取代（1.5.0 场景化首页）

## D9 · 场景化产品叙事首页（1.5.0）

- **决策**：从"案例驱动首页"（D8）升级为"场景化产品叙事首页"
- **取代**：D8（案例驱动首页）
- **原因**：
  - D8 的"主题+产物卡片"结构功能味太重，仍像"内容生成器"
  - 参考花间辞：先讲生活场景、用户痛点、产品闭环，再讲功能
  - btool 要表达成"古籍可信传播工作流"，不是"古籍内容生成器"
  - 四类用户（短视频/教师/文博/文创）需要差异化入口，而非通用主题输入
- **实现方式**：
  - 首屏=场景叙事（产品主张+真实用户任务+可信来源+生成预览+双CTA）
  - 场景入口=4 张场景卡，点击切换首屏任务文案和传播包产物组合
  - 指标区=6 项产品指标（可追溯率/标注率/演绎标注/4类覆盖/10s/异步）
  - 传播包=文字包+图片包+视频包+发布包+引用说明，5 段纵向堆叠
  - 下方=阅读器工作台（"进入创作"入口）
  - 保留：API配置/Bearer Token/异步轮询/XSS加固/主题输入/可信来源/阅读器/分享卡/移动端
- **日期**：1.5.0（2026-07-04）
- **状态**：已落地（被 D10 取代）

## D11 · 主案例优先 + 折叠传播包（1.8.0）

- **决策**：把 D10 案卷右栏从"5类产物平铺清单"改为"主案例预览卡优先 + 5类折叠传播包"，让首页第一眼更像完整传播案例
- **取代**：D10（案卷右栏5类产物平铺）的右栏信息层级
- **原因**：
  - D10 案卷右栏 5 类产物全部铺开，首屏像长清单，不像完整案例
  - 用户需要滚动很久才能看到全部内容，首屏信息密度低
  - "示例数据"标签在每张卡上反复出现，视觉噪音大
  - 用户进入首页的核心诉求是"看到一个完整的传播案例长什么样"，而非浏览分类清单
- **实现方式**：
  - 新增 `.hero-case-card` 主案例预览卡：封面图 + 口播稿标题/摘要 + 60秒分镜缩略（前3个）+ CTA 按钮，一眼呈现完整案例形态
  - 5 类传播包改为 `.pack-collapsible` 紧凑折叠行：文字/图片/视频/发布/引用，默认收起，点击标题行展开
  - CTA 按钮"生成完整传播包"一键展开全部 5 类
  - 移除单卡"示例数据"标签，仅底部 `apiStatus` 状态徽章保留说明
  - 首屏默认案卷高度 740px（不含主题区），远低于 1100px 目标
  - 900px 视口完整露出：原文来源 + 主案例产物 + CTA
  - 390px 移动端 hero-card 上下布局（单列），无横向溢出，首屏不被底部抽屉遮挡
  - GSAP 动画选择器迁移：入场时间线 + 场景切换 stagger 适配 `.hero-case-card` + `.pack-collapsible`
- **不新增功能**：不改 API 协议，不新增业务功能，纯信息层级与视觉表现升级
- **保留**：案卷整体结构（顶栏/左栏/底栏）、四类场景 tab、阅读器工作台、API 配置、移动端 peek 抽屉、分享卡下载等全部不变
- **日期**：1.8.0（2026-07-05）
- **状态**：已落地

## D10 · 案卷视觉收敛首页（1.7.0）

- **决策**：把 D1 工作台 / D8 案例区 / D9 场景叙事三块叠加的碎裂首页，收敛为单一"可信传播案卷"
- **取代**：D9（场景化产品叙事首页）
- **原因**：
  - D9 把场景叙事、场景入口大卡、指标条、传播包、工作台五块平铺，首页显得碎、模块堆叠
  - 企业 dashboard 式指标条与古籍气质冲突
  - 4 张大场景卡+emoji 图标让首页像功能展示页，不像案卷
  - 视觉语言混用：深色 hero + 纸色卡片 + 后台面板，不统一
- **实现方式**：
  - 单一 `<section class="case-file">` 案卷容器（水墨册页调性：纸色背景+SVG噪点纹理+册页双线边框+四角装饰）
  - 顶栏：案卷名 + 4 个轻量场景 tab（纯文字，无 emoji）+ 设置齿轮（API 配置入口）
  - 左栏卷宗：古籍原文（滚动）+ 来源 + 版本 + 原文链接 + "原"印章
  - 右栏产物：5 类纵向堆叠（口播稿/封面/分镜/发布标题/引用说明）
  - 底栏：来源声明文字（合并原 metrics 信息）+ "信"印章，非 dashboard 指标条
  - 标签收敛为 2 类：考据（绿）/ 演绎（朱砂），移除"可追溯"/"演示"等
  - 移除全部 emoji，改印章式文字标识
  - 统一卡片 border-radius 2px、border、padding、字体层级
  - API 配置默认隐藏，齿轮按钮触发
  - GSAP 选择器迁移到案卷结构，保留 matchMedia 三条件
- **不新增功能**：纯视觉收敛，后端能力/阅读器/分享卡/移动端 peek 全部保留
- **日期**：1.7.0（2026-07-04）
- **状态**：已落地
