## [2.0.0] 2026-07-05 · 摊开册页案台视觉重塑（纯视觉，不改业务逻辑）

> 本轮为纯视觉重塑，不新增功能、不改 API 协议、不改真实生成逻辑、不改阅读器工作台。目标是将"后台表单感"改为"摊开的古籍案台"气质，让用户第一眼感受到古籍传播案卷的沉浸感。

### 视觉重构
- **去硬边框**：全部 `border: 1px solid` / `border-top: 1px dashed` 替换为淡墨渐变分隔线（`linear-gradient(90deg, transparent, rgba(26,20,16,.1) 10%, ...)`）
- **纸张层次**：案台、主案例卡、案例卡采用三层 `box-shadow`（1px/4px/8px 渐进）替代矩形边框，叠加米色径向渐变模拟纸张纹理
- **折角与朱砂条**：主案例卡右上角三角折角（`border-width: 0 18px 18px 0`），左侧朱砂渐变条作题签
- **签条式折叠包**：`.pack-collapsible` 去边框、左朱砂竖条 `::before`、悬停加浓，从"后台列表"变"册页签条"
- **印章式 CTA**：主按钮采用 `ZCOOL XiaoWei` 字体、`letter-spacing: 3px`、`inset 0 0 0 2px rgba(255,255,255,.18)` 内描边、`border-radius: 2px`
- **场景标签 / 生成按钮**：统一 `ZCOOL XiaoWei` + 朱砂淡底，题签气质
- **主案例预览卡**：加金条 `::before`、纸纹叠加、折角，从占位卡变传播成品观感

### 保留（不回退）
- **1.9.0 功能结构不变**：hero-case-card + 5 类折叠传播包 + 案卷布局完全不动
- **API 逻辑不变**：方案 A LLM 直连、失败态保留 DOM、LLM/后端模式区分全部保留
- **真实文字生成不变**：`ApiClient._generateWithLLM()` 接入、JSON mode 输出、内容进入案例卡和传播包不变
- **阅读器工作台不变**：`#sidePanel` 完整保留，选文生成链路不受影响
- **移动端适配不变**：390px 无横向溢出、peek 抽屉延迟出现、首屏不被遮挡

### 验证（Playwright，3/3 通过）
- 桌面端：功能完整 + 去边框 + ZCOOL 字体 + 纸张阴影 + 签条式折叠包
- LLM 500 失败态：保留 hero-case-card + 5 pack-collapsible，文字包显示失败态 + 重试按钮
- 移动端 390px：无横向溢出，主 CTA 在首屏可见

### 已知限制
- 图片/视频仍为示例预览（未接入真实生成，符合 1.9.0 边界）
- 视觉重塑为 CSS 层面，未引入新的 JS 逻辑或 GSAP 动画

## [1.9.0] 2026-07-05 · 文字类真实生成接入（方案 A：前端 API Key 直连）

> 本轮为 P0 真实生成缺口推进，采用方案 A：前端 API Key 面板直连 OpenAI 兼容服务。停止首页视觉迭代，1.8.0 首页结构保持不变。

### 新增
- **文字类真实 LLM 生成**：`ApiClient._generateWithLLM()` 接入 OpenAI 兼容接口（/v1/chat/completions），支持 JSON mode 输出；生成内容自动适配现有 hero-case-card 和折叠传播包结构
- **模型可选配置**：API 配置面板新增"模型名称"输入框，默认 gpt-4o-mini，可自定义
- **API 类型自动识别**：BASE_URL 含 `/v1` 或配置了 Key 且不含 `/api/` 时自动走 LLM 直连模式，否则走原有后端 `/api/case/generate` 接口
- **安全提示**：配置面板增加醒目安全提示，说明 Key 仅存 localStorage，适合 demo/参赛，非生产级方案

### 保留
- **图片/视频四态结构不变**：未配置 / 生成中 / 失败 / 示例降级，状态标签和按钮逻辑全部保留，异步轮询结构不动
- **1.8.0 首页结构不变**：hero-case-card + 5 类折叠传播包 + 案卷布局完全不动，仅生成内容从 demo 切换为真实 LLM 输出
- **GSAP 动画边界**：仅案卷首屏入场和场景切换，不碰阅读器核心交互
- **阅读器选文生成**：完全保留，不受影响
- **html2canvas 分享卡下载**：完全保留
- **移动端适配**：完全保留

### 降级策略
- **无 BASE_URL / 无 Key**：直接走 `_fallbackCase` 示例数据，页面不坏
- **配置了 API 但请求失败**：显示失败态 + 错误信息 + 重试按钮，不静默降级
- **文字成功 / 图片视频 demo**：文字显示真实生成内容，图片视频保留示例预览状态

### 验证
- JS 语法检查通过
- 页面正常加载，无新增 console 错误
- hero-case-card / 5 类折叠传播包 / API 配置面板 UI 正常
- 无 API 配置时自动降级为示例数据
- API 配置面板新增模型输入框和安全提示正常显示

### 已知限制
- 图片/视频真实生成未接入，仍为示例预览 + API 预留结构
- 真实生成需用户自行配置 OpenAI 兼容服务的 API 地址和 Key
- API Key 存 localStorage，浏览器端有暴露风险，仅适合 demo/参赛场景

## [1.6.0] 2026-07-04 · GSAP 产品叙事动画（仅服务叙事，不碰阅读器）

### 新增
- **GSAP 引入**：`<head>` 加入 `gsap@3.12.5` CDN（jsdelivr 主源 + unpkg 兜底），双源保障；加载失败时降级为无动画，页面不坏
- **首屏入场时间线**：`gsap.timeline()` 编排"原文→来源→传播包"叙事节奏
  - 产品主张 → 真实用户任务 → 可信来源条 → 4 格生成预览 → 双 CTA → 场景入口 → 指标条 → 传播包
  - 仅用 `autoAlpha` / `y` / `scale`（transform+opacity），不动画 width/height/top/left
- **可信链路节点点亮**：`.hero-source-strip > *` 按 stagger 0.12s 顺序点亮（原文→来源→链接→可追溯），`back.out(1.6)` 微弹
- **场景切换传播包 stagger 过渡**：`switchScenario()` 和 `generateCaseWithTheme()` 末尾触发 `animatePackTransition()`，5 段 `.pack-section` 按 0.07s stagger 淡入

### 可访问性
- **prefers-reduced-motion 全程支持**：`gsap.matchMedia()` 三条件（isDesktop/isMobile/reduceMotion），reduce 模式下用 `gsap.set` 直接显示，无任何过渡
- **移动端轻量**：移动端位移和持续时间均减半（dy 12px / duration 0.45s）

### 边界
- **阅读器区域零干扰**：`.workspace` / `.shelf` / `.reader` 完全不动画，选中文字→工具条→生成产物链路不受影响
- **不装饰性炫技**：仅服务产品叙事三段（首屏入场/节点点亮/场景切换），其他区域无动效

### 修复
- **GSAP 守卫写法**：`typeof gsap!=='function'` 误判（GSAP 3 中 gsap 是对象），改为 `typeof gsap==='undefined'||typeof gsap.to!=='function'`
